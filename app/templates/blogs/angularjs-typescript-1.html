Part 1

Typescript with AngularJS and Node in VS Code.

By Alex Earley, Sunday November 19th, 2017. Updated Sunday November 26th, 2017.
It took me the better part of yesterday to build this small little 'Hello World' app. I am glad that I did. I wanted to introduce TypeScript to my stack, but not do it in the middle of a sprint at work. So I updated VS Code and installed TypeScript via NPM. I Google'd a handful of errors and questions, and soon enough I had a 'Hello World' alert popping up on ng-click.

Why TypeScript?

TypeScript is a superset of javascript. It compiles down to javascript. It introduces type saftey, interfaces, easier to read syntax, and real time error detection.
Step 1

Tools you will need.

  • Git Bash - https://git-scm.com/downloads
  • Visual Studio Code - https://code.visualstudio.com/
  • Node - https://nodejs.org/en/
Step 2

Install Node dependencies

In the Terminal Window type the following commands.
  • Press CTRL + '~' To Open up the Terminal. Choose bash.exe from the dropdown.
  • npm install -g typescript;
  • npm install -g yarn;
  • npm install -g uglify-es;
  • ( If you have uglifyJS installed, uninstall before installing uglifyES.)
To insure the tools installed correctly, run the following.
  • npm ls -g --depth 0;
Step 3

Add JS dependencies with yarn.

The following commands will add library dependencies to node_modules folder.
  • yarn add angular;
  • yarn add angular-route;
  • yarn add jquery;
Step 4

Setup your project

  • Create your root project folder and navigate to it with VS Code.
  • Create /app/
  • Create /app/app.ts
  • Create /index.html
  • Create /tsconfig.json
Step 5

Create a bundling script file

This is optional, you may run these commands however you'd like. I prefer to dump them into a bundle.sh file and run that.
  • Create bundle.sh in root directory, paste the following.
  • tsc;
uglifyjs \
./node_modules/angular/angular.min.js \
./node_modules/angular-route/angular-route.min.js \
./app/app.js \ -o
./main.min.js \ --source-map url=main.min.js.map;
Step 6

Definitely Typed - TypeScript type definitions

You may ask yourself, if my code is in TypeScript then don't my libraries need to be .ts files? TypesScript compiles down to JS, and your JS will look like a normal angular app that has the same normal angular dependecies. In order to have your TypeScript code correctly integrate with your libraries you do need to have whats called 'Typed Definitions'. These help TypeScript interface with common JS libraries.
We can install these using yarn.
  • yarn add @types/angular;
  • yarn add @types/angular-route;
  • yarn add @types/jquery;

Bringing it all together

Finally we will need to setup tsconfig.json and index.html files. The tsconfig will tell TypeScript Compiler (tsc) what files it needs to compile and tells VS Code what type definitions to use. Because the bundle.sh comiles everything into a single main.min.js, the index file will only need one JS reference.
Step 7

tsconfig.json

{
    "compilerOptions": {
        "sourceMap": true,
        "target": "es6",
        "module":"commonjs",
        "allowSyntheticDefaultImports": true,
        "types": [
            "angular",
            "angular-route",
            "jquery"
        ]
    },
    "files": [
        "node_modules/@types/angular/index.d.ts",
        "node_modules/@types/angular-route/index.d.ts",
        "node_modules/@types/jquery/index.d.ts",
        "app/app.ts"
    ]
}

What do we have so far?

We have a bunch of useless files and a bundle.sh that does something. But no 'Hello World' yet. Now that we are almost done with set up, time to learn some more about AngularJS.
